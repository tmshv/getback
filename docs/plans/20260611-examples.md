# Examples: one-sheep, several-sheep, only-corgi Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add three runnable example packages — `examples/one-sheep`, `examples/several-sheep`, and `examples/only-corgi` — each demonstrating a focused slice of the simulation. Every example reuses `@getback/game` in full (assets, render, input, run-loop); the only bespoke code per example is a small, headless-testable `buildWorld(seed)` factory and a three-line `main.ts`. A shared `runScenario(world)` helper exported from `@getback/game` encapsulates the DOM setup so the examples stay readable and never drift apart.

**Architecture:** Each example is a thin npm-workspace package (`examples/<name>/`) with its own `package.json`, `index.html`, `vite.config.ts`, and `src/main.ts`. `main.ts` calls `buildWorld(seed)` then `runScenario(world)`. The scenario factory is the only testable surface; Vitest runs it headless. The Vite dev server boot and visual correctness of each example are verified manually. `@getback/game` gains one new export — `runScenario(world, opts?)` — that wires keyboard input via `createInput()` and calls `mount(world, { input: input.intent.bind(input), ...opts })` (world FIRST, opts SECOND, per Plan 16's canonical signature). This is chosen over duplicating the wiring boilerplate in every example because: (a) the `buildWorld` factory is the pedagogically interesting part — readers learn the motor API; (b) a single `runScenario` helper is two lines, impossible to get wrong, and prevents three copies of input-wiring code drifting apart over time.

**Tech Stack:** TypeScript 5 strict, ESM, `.js` import extensions on `.ts` source files. Vitest 2 (`npx vitest run <path>` per file, `npm test` suite, `npm run typecheck`). Vite dev server per example. `@getback/game` (mount, runScenario, HUD auto-hides), `@getback/motor` (createWorld, createDog, createSheep, rollSheepTraits, defaultSheepTraits, buildPen, generatePen, createGrassField, config, makeRng). One-line imperative commits per task.

## Key facts

- `createWorld(sheep?, grass?, obstacles?, pen?, dog?, rng?)` — all parameters optional; `pen: null` and `dog: null` are valid. Exported from `@getback/motor`.
- `createDog(pos: Vec2): Dog` — place at centre of bounds. Exported from `@getback/motor`.
- `createSheep(pos: Vec2, traits: SheepTraits): Sheep` — position + pre-rolled traits. Exported from `@getback/motor`.
- `rollSheepTraits(rng: Rng): SheepTraits` — varies around config baseline with ±20% jitter; added in Plan 14 (spawn-infra). Exported from `@getback/motor`.
- `defaultSheepTraits(): SheepTraits` — baseline traits, no jitter. Exported from `@getback/motor`.
- `generatePen(rng, opts): PenShape` + `buildPen(outline, gateEdge): Pen` — compose to make a pen. Exported from `@getback/motor`.
- `config.pen` — `{ rMin: 40, rMax: 60, minVerts: 5, maxVerts: 9, minGateWidth: 24 }`.
- `config.bounds` — `{ x: 0, y: 0, w: 480, h: 270 }`. Dog spawn centre: `{ x: 240, y: 135 }`.
- `createGrassField(opts): GrassField` — imported from `@getback/motor`. The default grass used inside `createWorld` is good; examples may pass `undefined` to accept the default.
- `mount(world, opts?)` — exported from `@getback/game` (Plan 16). `opts.input?: () => DogIntent` feeds intent each tick; `opts.container?: HTMLElement` sets the canvas target; `opts.hud?: HudOverride` (added in Plan 17) controls HUD visibility.
- `createInput(): { intent(): DogIntent; dispose(): void }` — keyboard input wiring exported from `@getback/game` (Plan 17).
- `runScenario(world, opts?)` — **new** export added to `@getback/game` in Task 1 of this plan. Calls `createInput()`, then `mount(world, { input: input.intent.bind(input), ...opts })` (implemented in Plans 16–17). Returns `void`.
- HUD auto-hides the flock counter when `world.pen === null` and `world.sheep.length === 0` (spec §13.3). No per-example HUD override needed; the default detection covers all three scenarios correctly.
- The root `package.json` workspace glob is `"examples/*"` — already present; no root changes needed.
- `vitest.config.ts` includes `"packages/**/*.test.ts"` — example tests live under `examples/*/src/`, so they need their own `vitest.config.ts` per package, OR the root config must be broadened. **Chosen:** broaden the root `vitest.config.ts` include to `["packages/**/*.test.ts", "examples/**/*.test.ts"]` so a single `npm test` runs everything.
- Sheep scatter positions: use `rng.range(margin, w - margin)` with `margin = 30` and `w/h` from `config.bounds` to keep sheep inside the pasture.
- `makeRng(seed: number): Rng` from `@getback/math` — gives a seeded, deterministic RNG.

- [ ] **Step 3: Create `examples/one-sheep/src/world.ts`**

```ts
import {
  createWorld,
  createDog,
  createSheep,
  rollSheepTraits,
  generatePen,
  buildPen,
  penContains,
  config,
} from "@getback/motor";
import { makeRng } from "@getback/math";
import type { World } from "@getback/motor";

// Builds a minimal herding scenario: one sheep, a dog at centre, and a small
// randomly-generated pen. The sheep starts scattered outside the pen. The
// caller passes a seed for determinism (headless tests) or Date.now() for
// variety in the browser.
export function buildWorld(seed: number): World {
  const rng = makeRng(seed);
  const b = config.bounds;
  const margin = 30;

  // pen: small, near the bottom-right quadrant so the sheep has room to roam
  const penCenter = {
    x: rng.range(b.x + b.w * 0.55, b.x + b.w - config.pen.rMax - margin),
    y: rng.range(b.y + b.h * 0.55, b.y + b.h - config.pen.rMax - margin),
  };
  const shape = generatePen(rng, {
    center: penCenter,
    rMin: config.pen.rMin,
    rMax: config.pen.rMax,
    minVerts: config.pen.minVerts,
    maxVerts: config.pen.maxVerts,
    minGateWidth: config.pen.minGateWidth,
  });
  const pen = buildPen(shape.outline, shape.gateEdge);

  // sheep: scatter in the top-left half, outside the pen
  let sheepPos = { x: rng.range(b.x + margin, b.x + b.w * 0.45), y: rng.range(b.y + margin, b.y + b.h * 0.45) };
  for (let tries = 0; tries < 20 && penContains(pen, sheepPos); tries++) {
    sheepPos = { x: rng.range(b.x + margin, b.x + b.w - margin), y: rng.range(b.y + margin, b.y + b.h - margin) };
  }
  const traits = rollSheepTraits(rng);
  const sheep = createSheep(sheepPos, traits);

  // dog: centre of the pasture
  const dog = createDog({ x: b.x + b.w / 2, y: b.y + b.h / 2 });

  return createWorld([sheep], undefined, [], pen, dog, rng);
}
```

- [ ] **Step 3: Create `examples/several-sheep/src/world.ts`**

```ts
import {
  createWorld,
  createDog,
  createSheep,
  rollSheepTraits,
  generatePen,
  buildPen,
  penContains,
  config,
} from "@getback/motor";
import { makeRng } from "@getback/math";
import type { World } from "@getback/motor";

const SHEEP_COUNT = 5;

// Builds a small flocking scenario: five sheep with varied traits, a dog at
// centre, and a randomly-generated pen. Each sheep is placed outside the pen.
// Shows cohesion, follow, and fear-bunching at small flock scale.
export function buildWorld(seed: number): World {
  const rng = makeRng(seed);
  const b = config.bounds;
  const margin = 30;

  // pen: right-centre of the pasture
  const penCenter = {
    x: rng.range(b.x + b.w * 0.6, b.x + b.w - config.pen.rMax - margin),
    y: rng.range(b.y + config.pen.rMax + margin, b.y + b.h - config.pen.rMax - margin),
  };
  const shape = generatePen(rng, {
    center: penCenter,
    rMin: config.pen.rMin,
    rMax: config.pen.rMax,
    minVerts: config.pen.minVerts,
    maxVerts: config.pen.maxVerts,
    minGateWidth: config.pen.minGateWidth,
  });
  const pen = buildPen(shape.outline, shape.gateEdge);

  // sheep: scatter across the left half, all outside the pen
  const sheep = [];
  for (let i = 0; i < SHEEP_COUNT; i++) {
    let pos = {
      x: rng.range(b.x + margin, b.x + b.w * 0.5),
      y: rng.range(b.y + margin, b.y + b.h - margin),
    };
    for (let tries = 0; tries < 20 && penContains(pen, pos); tries++) {
      pos = {
        x: rng.range(b.x + margin, b.x + b.w - margin),
        y: rng.range(b.y + margin, b.y + b.h - margin),
      };
    }
    sheep.push(createSheep(pos, rollSheepTraits(rng)));
  }

  const dog = createDog({ x: b.x + b.w / 2, y: b.y + b.h / 2 });

  return createWorld(sheep, undefined, [], pen, dog, rng);
}
```

import { createWorld, createDog, config } from "@getback/motor";
import { makeRng } from "@getback/math";
import type { World } from "@getback/motor";

// Builds a dog-only scenario: no sheep, no pen. Used to tune movement feel,
// sprint responsiveness, bark FX, and stamina drain/regen without the
// complexity of herding. The HUD flock counter auto-hides (spec §13.3).
export function buildWorld(seed: number): World {
  const rng = makeRng(seed);
  const b = config.bounds;
  const dog = createDog({ x: b.x + b.w / 2, y: b.y + b.h / 2 });
  return createWorld([], undefined, [], null, dog, rng);
}
